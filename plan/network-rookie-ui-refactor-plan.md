# Network Rookie — UI Refactor Plan

## Mission

Transform Network Rookie from a visually noisy prototype into a calm, professional educational visualization product.

This refactor focuses on:
- visual consistency
- theme unification
- typography hierarchy
- realism
- restraint
- beginner readability

This is a VISUAL REFACTOR ONLY.

No major architecture rewrites.
No feature expansion.

---

# Core Problems

Current UI problems:

- cyberpunk visual noise
- too many accent colors
- excessive glow usage
- inconsistent theming
- overloaded telemetry
- weak typography hierarchy
- too many emoji affordances
- conflicting theme systems
- feels AI-generated instead of intentional

---

# Desired Product Feeling

The product should feel:

- elegant
- calm
- educational
- modern
- premium
- trustworthy
- exhibition-ready

Primary references:
- Linear
- Vercel
- Apple developer tooling

---

# Refactor Scope

## Included

- theme refactor
- visual cleanup
- typography refinement
- spacing refinement
- globe visual redesign
- route visualization redesign
- control redesign
- animation polish
- dark/light mode cleanup

## Excluded

- backend work
- AI logic changes
- architecture rewrites
- new major features
- simulation engine changes

---

# Refactor Phases

## Phase 1 — Theme System Cleanup

### Goals

Unify all theme systems.

### Tasks

- remove duplicated tokens
- remove figma-generated leftovers
- simplify semantic variables
- unify dark/light mode structure
- reduce accent usage

### Acceptance Criteria

- one coherent token system
- no duplicated color meanings
- easier theme maintenance

---

## Phase 2 — Typography & Layout

### Goals

Improve readability and visual hierarchy.

### Tasks

- migrate toward clean sans-serif typography
- reduce mono usage
- improve spacing rhythm
- simplify card hierarchy
- reduce visual density

### Acceptance Criteria

- interface feels breathable
- typography guides attention naturally
- beginner readability improved

---

## Phase 3 — Globe Redesign

### Goals

Create realistic earth-inspired visuals.

### Tasks

- realistic ocean colors
- natural land palettes
- desert color zones
- restrained atmosphere
- improved globe contrast

### Acceptance Criteria

- globe feels believable
- readable in dark/light mode
- no cyberpunk feeling

---

## Phase 4 — Route Visualization Redesign

### Goals

Make routes elegant and understandable.

### Tasks

- thinner route arcs
- subtle pulse animation
- remove chaotic particles
- reduce glow intensity
- improve hierarchy clarity

### Acceptance Criteria

- routes readable instantly
- motion feels calm
- less visual noise

---

## Phase 5 — Chat & Control Simplification

### Goals

Modernize the right panel.

### Tasks

- redesign simulation controls
- remove emoji-heavy UI
- simplify chat chips
- improve spacing
- reduce borders/shadows

### Acceptance Criteria

- controls feel professional
- chat feels modern and minimal
- easier to scan quickly

---

## Phase 6 — HUD & Overlay Polish

### Goals

Reduce visual competition.

### Tasks

- simplify telemetry visuals
- reduce brightness
- improve transparency handling
- minimize overlay dominance

### Acceptance Criteria

- globe remains visual hero
- overlays feel secondary
- cleaner presentation overall

---

# Final Success Criteria

The finished application should:

- feel intentional
- feel professionally designed
- avoid “AI generated” aesthetics
- feel calm and educational
- remain beginner friendly
- look exhibition ready

A first-time user should immediately understand:
- where to look
- what matters
- what the visualization is doing

without being visually overwhelmed.
